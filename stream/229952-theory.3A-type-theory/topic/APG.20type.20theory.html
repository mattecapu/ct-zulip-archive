Looking at the whole picture:

  1. You have a schema SSS, which is a pair (L,σ)(\mathcal{L}, \sigma)(L,σ). L\mathcal{L}L is a set of labels, and σ\sigmaσ assigns each label to a type. Thus, a schema is a bunch of typed labels. Your labels could be “Car”, “Elizabeth”, “meets”. The only types are unit, sum, product, primitive, and label; so I guess σ\sigmaσ might assign individuals to 111, essential concepts to “primitive”, any sort of function, relation or grouping to a product type, any sort of “optional grouping” to a sum type, and I’m not sure what the label type is doing in there, since the labels should be in L\mathcal{L}L, whereas all these other typed values should be in V\mathcal{V}V.

  1. You have elements.

  2. You map elements to their labels.

  3. You map values to their types.

  4. You map elements to their values.

And I guess the takeaway here is that our labels have type annotations, and our values have type annotations, and every element has a label and a value.
