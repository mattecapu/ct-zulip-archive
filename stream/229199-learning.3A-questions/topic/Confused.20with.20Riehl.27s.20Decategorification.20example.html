So in what sense are the products from Set getting carried over?

This is how decategorification works:

Any functor F:C→DF : C \to DF:C→D between essentially small categories gives a function

Decat(F):Decat(C)→Decat(D)\mathrm{Decat}(F) : \mathrm{Decat}(C) \to \mathrm{Decat}(D)Decat(F):Decat(C)→Decat(D)

where Decat(C)\mathrm{Decat}(C)Decat(C) is the set of isomorphism classes of objects of CCC and similarly for DDD. Here's how it works:

Decat(F)([c])=[d]\mathrm{Decat}(F)([c]) = [d] Decat(F)([c])=[d]

where [c][c][c] is the isomorphism class of the object c∈Cc \in Cc∈C, and similarly for ddd.

I think from this you can figure out how decategorifying the product functor

×:FinSet×FinSet→FinSet \times : \mathsf{FinSet} \times \mathsf{FinSet} \to \mathsf{FinSet}×:FinSet×FinSet→FinSet

we get the usual multiplication of natural numbers, which is

Decat(×):N×N→N \mathrm{Decat}(\times) : \mathbb{N} \times \mathbb{N} \to \mathbb{N}Decat(×):N×N→N

How does that work?

You can collapse any category CCC to a preorder Pre(C)\mathrm{Pre}(C)Pre(C) whose elements are objects of CCC and c≤c′c \le c'c≤c′ iff there's a morphism f:c→c′f: c \to c'f:c→c′.

Any functor F:C→DF: C \to DF:C→D then gives a monotone map, let's call it

Pre(F):Pre(C)→Pre(D)\mathrm{Pre}(F): \mathrm{Pre}(C) \to \mathrm{Pre}(D)Pre(F):Pre(C)→Pre(D),

in an evident way. We can show Pre\mathrm{Pre}Pre thus defined is a functor from the category of categories to the category of preorders.

But if there's a natural transformation α:F→G\alpha : F \to Gα:F→G we don't usually have Pre(F)=Pre(G)\mathrm{Pre}(F) = \mathrm{Pre}(G)Pre(F)=Pre(G). What we instead get is

Pre(F)≤Pre(G)\mathrm{Pre}(F) \le \mathrm{Pre}(G)Pre(F)≤Pre(G)

Here I'm using the fact that given two monotone maps between preorders, say f,g:P→Qf, g: P \to Qf,g:P→Q, we can define f≤gf \le gf≤g to mean

∀p∈Pf(p)≤g(p)\forall p \in P \quad f(p) \le g(p)∀p∈Pf(p)≤g(p)

Using this we see that there's not just a set of monotone maps between preorders, but a preorder of them.

And using this we get a 2-category Preord\mathbf{Preord}Preord of preorders, monotone maps, and ≤\le ≤ relations between monotone maps!

John Baez (Oct 03 2024 at 01:53):

I can't resist saying a bit more. The category Set\mathsf{Set}Set has a baby sister called Boole\mathsf{Boole}Boole with just two objects, FFF and TTT. There's a functor Set→Boole\mathsf{Set} \to \mathsf{Boole}Set→Boole that sends empty sets to FFF and nonempty ones to TTT. This functor is symmetric monoidal in two ways: it sends coproduct to "or" and product to "and".

In enriched category theory, a Set\mathsf{Set}Set-enriched category is just a category, since it has a set of morphisms from any object to any other. A Boole\mathsf{Boole}Boole-enriched category is a preorder, since it has a truth value of morphisms from any object to any other.

The functor Set→Boole\mathsf{Set} \to \mathsf{Boole}Set→Boole I mentioned can be used to do base change. In this example, base change turns any Set\mathsf{Set}Set-enriched category to a Boole\mathsf{Boole}Boole-enriched category, by turning the hom-sets into hom-truth values. And this base change process is just what I've been calling Pre:Cat→Preord\mathrm{Pre} : \mathbf{Cat} \to \mathbf{Preord}Pre:Cat→Preord.

John Baez (Oct 03 2024 at 01:57):

This is why Pre\mathrm{Pre}Pre is so beautiful: this process for turning categories into preorders emerges automatically from the simpler process of turning sets into truth values.
