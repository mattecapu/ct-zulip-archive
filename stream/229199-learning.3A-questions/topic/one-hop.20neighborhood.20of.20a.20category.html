Topic: one-hop neighborhood of a category

Bruno Gavranović (Mar 16 2022 at 14:04):

Something I realised I can do with graphs, but not with categories, is to talk about the "one-hop neighbourhood" of a node.

Say I have the graph below. image.png

I can ask "What are the nodes neighbouring V1V_1V1​?". The answer is (V0,V1,V2,V3)(V_0, V_1, V_2, V_3)(V0​,V1​,V2​,V3​) . The node V4V_4V4​ is not a neighbour since it takes two hops to get to V4V_4V4​.

But if I consider this graph as a presentation of a category, then I lose the ability to talk about "one-hop" neighbourhoods. Since V0V_0V0​ is a neighbour of V1V_1V1​ (via V1→e0V0V_1 \xrightarrow{e_0} V_0V1​e0​​V0​) and V4V_4V4​ is a neighbour of V0V_0V0​ (via V0→e5V4V_0 \xrightarrow{e_5} V_4V0​e5​​V4​), then V0V_0V0​ is a "neighbour" of V4V_4V4​ (via the composite V1→e0;e5V4V_1 \xrightarrow{e_0 ; e_5} V_4V1​e0​;e5​​V4​. That is, any object is connected to any other object, as long as there is a sequence of morphisms connecting them.

Is it possible to model the idea of one-hop neighborhood with categories? Or do I relinquish this possibility by moving from graphs to categories?

Spencer Breiner (Mar 16 2022 at 17:04):

How about this: Write N\NN for the one object category, and its underlying graph. Any graph GGG has a canonical homomorphism G→NG\to\NG→N sending every edge to 1. Hitting this with the free/forgetful adjunction, this is the same as a functor FG→NFG\to\NFG→N.

Now if 2={0→1}\mathbf{2}=\{0\to 1\}2={0→1} is the "walking arrow", the one-hop paths are represented by the pullback

∙→FG↓↓2→1N\begin{CD} \bullet @>>> FG\\ @VVV @VVV\\ \mathbf{2} @>>1> \N \end{CD}∙↓⏐​2​​1​​FG↓⏐​N​

You can see that this prevents composition of edges by pairing them with the unique arrow in 2\mathbf{2}2.

Nathaniel Virgo (Mar 18 2022 at 00:47):

I guess you could define the one-hop neighbourhood of an object following this definition that Jules gave:

Jules Hedges said:

an edge is a morphism that has no non-trivial factorisation

Just define the one-hop neighbourhood of CCC as the set of objects that can be reached from CCC by an edge. (Though maybe it makes more sense to define it as the set of edges itself, since there might be more than one edge between the same two objects.)

This works in any category, not just free ones, but it might not always be useful. Consider the reals as a preorder. In that category every non-identity morphism has a non-trivial factorisation, so the one-hop neighbourhood of any object is empty (apart from the object itself, via its identity morphism). But maybe that makes sense.
