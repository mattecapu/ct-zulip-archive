The question is pretty well laid out on the link, but the main idea is: why does an enriched natural transformation use maps from the monoidal unit III, as opposed from some object P:VP : \mathcal{V}P:V?

Using objects that are different from the monoidal unit seems to be in line what enriched CT is all about, and the whole reasoning of using weighted limits. But Kelly's enriched CT book (and all other sources I could find) define it as a map from the monoidal unit.

Matteo Capucci (he/him) (Nov 25 2020 at 12:54):

I want to point out that using III is not necessary to express naturality. Well, it is if you do it the way Kelly does, using the unitors of V\mathcal VV, but it is not if you just write down the equation expressing naturality.
What I mean here is that given two V\mathcal VV-functors F,G:C→DF, G : \mathbf C \to \mathbf DF,G:C→D, one can define a V\mathcal VV-natural transformation α:F→G\alpha:F \to Gα:F→G to be an obCob \mathbf CobC-indexed collection of morphisms αX:P→D(FX,GX)\alpha_X : P \to \mathbf D(FX, GX)αX​:P→D(FX,GX).
Then you ask naturality in this way: given f:Q→C(X,Y)f : Q \to \mathbf C(X,Y)f:Q→C(X,Y), you want

αX⊗(f;GX,Y);∘D=((f;FX,Y⊗αY);∘D\alpha_X \otimes (f;G_{X,Y}) ; \circ_D = ((f;F_{X,Y} \otimes \alpha_Y) ; \circ_DαX​⊗(f;GX,Y​);∘D​=((f;FX,Y​⊗αY​);∘D​

Matteo Capucci (he/him) (Nov 25 2020 at 13:04):

Fixing PPP for all the αX\alpha_XαX​ seems unavoidable, since there would be no way to express naturality then (basically, the left and right hand of the above equation would have a different source!)
To relax this, one could go galaxy brain and use the end definition of natural transformations:
[F,G]=∫X∈CD(FX,GX)[F,G] = \int_{X \in \mathbf C} \mathbf D(FX,GX)[F,G]=∫X∈C​D(FX,GX)
Then one would add a weight W:C→VW : \mathcal C \to \mathcal VW:C→V and define WWW-weighted natural transformations to be
[F,G]W=∫X∈C[WX,D(FX,GX)][F,G]^W = \int_{X \in \mathbf C} [WX, \mathbf D(FX,GX)][F,G]W=∫X∈C​[WX,D(FX,GX)]

@Matteo Capucci (he/him) and @Nathanael Arkor: I finally figured out how this works! I think we indeed can't define DiNatV(F,G)\mathbf{DiNat}_{\mathcal{V}}(F,G)DiNatV​(F,G) in general: there seems to be a kind of microcosm principle going on here, and the answer is to enrich in CoMon(V)\mathsf{CoMon}(\mathcal{V})CoMon(V) instead:

  • First, the reason this is not an issue in ordinary category theory is because CoMon(V)≅V\mathsf{CoMon}(\mathcal{V})\cong\mathcal{V}CoMon(V)≅V for V\mathcal{V}V Cartesian monoidal, and hence CatsCoMon(V)≅CatsV\mathsf{Cats}_{\mathsf{CoMon}(\mathcal{V})}\cong\mathsf{Cats}_{\mathcal{V}}CatsCoMon(V)​≅CatsV​ also, so CoMon(Sets)\mathsf{CoMon}(\mathsf{Sets})CoMon(Sets)-enriched categories are just plain categories.
  • Second, the comultiplication maps on the Homs: they are exactly what you need to define DiNatV(F,G)\mathbf{DiNat}_{\mathcal{V}}(F,G)DiNatV​(F,G) (the comonoid axioms are used to show that it is functorial, for instance);

  • Second+1/2: There's a kind of "loss of Ek\mathbb{E}_{k}Ek​-ness" going on for dinaturality (like taking categories of monoids of ordinary, braided, or symmetric monoidal categories gives a category, a monoidal category, or a symmetric monoidal category respectively):
    - For V\mathcal{V}V-enrichment, DiNatV(F,G)\mathbf{DiNat}_{\mathcal{V}}(F,G)DiNatV​(F,G) can't be defined;
    - For CoMon(V)\mathsf{CoMon}(\mathcal{V})CoMon(V)-enrichment, DiNatV(F,G)\mathbf{DiNat}_{\mathcal{V}}(F,G)DiNatV​(F,G) is an object of V\mathcal{V}V;
    - For CCoMon(V)\mathsf{CCoMon}(\mathcal{V})CCoMon(V)-enrichment, DiNatV(F,G)\mathbf{DiNat}_{\mathcal{V}}(F,G)DiNatV​(F,G) is a comonoid in V\mathcal{V}V (and happens to be also a cocommutative one for enriched categories, but this won't happen at this level of cocommutativity for enriched ∞\infty∞-categories).
    So F,G↦DiNatV(F,G)F,G\mapsto\mathbf{DiNat}_{\mathcal{V}}(F,G)F,G↦DiNatV​(F,G) gives a functor from pairs of functors enriched in Ek\mathbb{E}_kEk​-coalgebras in V\mathcal{V}V to Ek−1\mathbb{E}_{k-1}Ek−1​-coalgebras in V\mathcal{V}V, where "E0\mathbb{E}_{0}E0​" corresponds to just an object in V\mathcal{V}V, E1\mathbb{E}_{1}E1​ to a comonoid, and E2=E3=⋯\mathbb{E}_{2}=\mathbb{E}_{3}=\cdotsE2​=E3​=⋯ to a cocommutative comonoid.

  • Third: the counit maps turn out to be exactly what you need to define constant V\mathcal{V}V-functors!

The reason the "reduction of cocommutativity" appears is because of Eckmann–Hilton: given V\mathcal{V}V-functors F,G ⁣:Cop⊠VC⇉DF,G\colon\mathcal{C}^{\mathsf{op}}\boxtimes_{\mathcal{V}}\mathcal{C}\rightrightarrows\mathcal{D}F,G:Cop⊠V​C⇉D, we'd like to define DiNatV(F,G)\mathbf{DiNat}_{\mathcal{V}}(F,G)DiNatV​(F,G) as the V\mathcal{V}V-end of the assignment

(A,B)↦HomD(FAB,GBA).(A,B)\mapsto\mathbf{Hom}_{\mathcal{D}}(F^{B}_{A},G^{A}_{B}).(A,B)↦HomD​(FAB​,GBA​).

It is here that the reduction of cocommutativity appears: for ΔC\Delta_{\mathcal{C}}ΔC​ to exist, we need an enrichment in CoMon(V)\mathsf{CoMon}(\mathcal{V})CoMon(V), and for it to be a CoMon(V)\mathsf{CoMon}(\mathcal{V})CoMon(V)-functor, we need an enrichment in CCoMon(V)\mathsf{CCoMon}(\mathcal{V})CCoMon(V): being CoMon(V)\mathsf{CoMon}(\mathcal{V})CoMon(V)-functorial means in particular that the comultiplication morphisms HomC(A,B)→HomC(A,B)⊗VHomC(A,B)\mathbf{Hom}_{\mathcal{C}}(A,B)\to\mathbf{Hom}_{\mathcal{C}}(A,B)\otimes_{\mathcal{V}}\mathbf{Hom}_{\mathcal{C}}(A,B)HomC​(A,B)→HomC​(A,B)⊗V​HomC​(A,B) are morphisms of comonoids, and this is is equivalent to the Hom\mathbf{Hom}Hom- objects being cocommutative, by Eckmann–Hilton.
