Topic: Combinations.

Julius Hamilton (Jan 12 2025 at 02:24):

Imagine there are 2 options for 3 independent things.

You could have (a,a,a)(a, a, a)(a,a,a), (a,b,a)(a, b, a)(a,b,a), (b,b,a)(b, b, a)(b,b,a), etc.

I claim there are 2×2×2=23=82 \times 2 \times 2 = 2^3 = 82×2×2=23=8 possible versions of such a choice.

For 888 things, I claim there are 8!=403208! = 403208!=40320 ways to put them in an order, because there are 888 choices for the first position, 777 for the second, etc.

Some of these orderings appear methodical. For example:

(a,a,a),a,a,b),(a,b,a),(a,b,b),(a, a, a), a, a, b), (a, b, a), (a, b, b),(a,a,a),a,a,b),(a,b,a),(a,b,b),
(b,a,a),(b,a,b),(b,b,a),(b,b,b)(b, a, a), (b, a, b), (b, b, a), (b, b, b)(b,a,a),(b,a,b),(b,b,a),(b,b,b) is an order that seems like it can be described by a rule. But other orders may appear random.

What theory analyzes which orderings are describable by a rule of some kind? Of the number of possible orderings, is there a number of orderings which are the result of a rule?

Peva Blanchard (Jan 12 2025 at 07:37):

It is not specifically about orderings, but algorithmic information theory deals with this kind of question. Especially, the Kolmogorov complexity of a binary sequence x=1100101001x = 1100101001x=1100101001 is defined, roughly speaking, as the length K(x)K(x)K(x) of the shortest computer program that can generate that sequence.

The sequence xxx is said to be random when K(x)K(x)K(x) is about the same size as the sequence xxx itself. Intuitively, this says that there is not a better computer program than the one that just prints out the sequence.
