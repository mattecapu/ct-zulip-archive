Dan Doel (Sep 13 2020 at 17:02):

There is a type ΩΩΩ whose values are propositional types. This allows you to have isCat:Animal→Ω\mathsf{isCat} : \mathsf{Animal} → ΩisCat:Animal→Ω act in a sort of double way. It's an arrow into the subobject classifier, but it's also a family of types. This happens a lot if you decide to make the type theory have types whose values are other types. isCat\mathsf{isCat}isCat is both a map into the subobject classifier and a family of types, while in topos theory there is just a correspondence between these two things.

Dan Doel (Sep 13 2020 at 17:17):

Since isCat\mathsf{isCat}isCat is a family of types indexed by Animal\mathsf{Animal}Animal, we can form the sigma type Σa:AnimalisCat(a)Σ_{a:\mathsf{Animal}} \mathsf{isCat}(a)Σa:Animal​isCat(a). There are generally multiple ways to think about this. If you think about the family BBB as primary, it's the disjoint union of all the fibers, or the sort of 'total space' of the family. In category theory you usually think of the family as already being the total space with a display map into the indexing type, but type theory prefers to present the family like it's mapping from the indexing type.

But, since the fibers of isCat\mathsf{isCat}isCat are propositional, this can also be read like it's a subset of Animal\mathsf{Animal}Animal, {c∈Animal∣isCat(c)}\{c \in \mathsf{Animal} | \mathsf{isCat}(c) \}{c∈Animal∣isCat(c)}. The values of the sigma type are animals together with proofs that they are cats, but there is 'at most one' such proof (homotopically if you're doing that) for each animal. So the values of the sigma type are more structured than the elements of a subset are usually thought to be, but the structure is in some sense just acting as a filter.

Dan Doel (Sep 13 2020 at 18:02):

If you want to get really into the weeds, then there is an alternate way of presenting this. You have a type ΩΩΩ of propositions, and a family

p:Ω⊢True(p) typep : Ω ⊢ \mathsf{True}(p)\ \mathsf{type}p:Ω⊢True(p) type

of truth values. Pullback corresponds to substitution, so if you have

a:Animal⊢isCat(a):Ωa : \mathsf{Animal} ⊢ \mathsf{isCat}(a) : Ωa:Animal⊢isCat(a):Ω

then the substitution

a:Animal⊢True(isCat(a)) typea : \mathsf{Animal} ⊢ \mathsf{True}(\mathsf{isCat}(a))\ \mathsf{type}a:Animal⊢True(isCat(a)) type

is the family you get by pulling back the truth family along the arrow isCat:Animal→Ω\mathsf{isCat} : \mathsf{Animal} → ΩisCat:Animal→Ω.

Dan Doel (Sep 13 2020 at 18:03):

Then you can use Σ to realize the total space of the family as a type in itself, rather than a family of types.

Dan Doel (Sep 13 2020 at 18:06):

This doesn't blur the lines of what you do in the category theory as much.

Dan Doel (Sep 14 2020 at 22:24):

The point of (sub)object classifiers is that there are correspondences between (certain) P→AP → AP→A and A→UA → \mathcal UA→U or A→ΩA → ΩA→Ω. The 'in the weeds' presentation I talked about makes the fact that this is a correspondence more explicit, but other times they are presented as just being the same thing.

So, you could hypothesize a family

a:Animal⊢isCat(a) propa : \mathsf{Animal} ⊢ \mathsf{isCat}(a)\ \mathsf{prop}a:Animal⊢isCat(a) prop

Meaning essentially that there is a monomorphism isCat:Cat↣Animal\mathsf{isCat} : \mathsf{Cat} \rightarrowtail \mathsf{Animal}isCat:Cat↣Animal, which is also an object of the slice category over Animal\mathsf{Animal}Animal. The obvious issue is that I have already invoked a type Cat\mathsf{Cat}Cat because I don't know what else to call that part of the subobject.

ΩΩΩ being a subobject classifier means that there exists isCat~:Animal→Ω\widetilde{\mathsf{isCat}} : \mathsf{Animal} → ΩisCat:Animal→Ω, which forms part of a pullback square with isCat:Cat↣Animal\mathsf{isCat} : \mathsf{Cat} \rightarrowtail \mathsf{Animal}isCat:Cat↣Animal. The other parts of the square are true:1↣Ω\mathsf{true} : 1 \rightarrowtail Ωtrue:1↣Ω and !:Cat→1! : \mathsf{Cat} → 1!:Cat→1. isCat~\widetilde{\mathsf{isCat}}isCat maps animals to the truth value that corresponds to their cat-hood.

In the more explicit notation I gave, the expectation is something like:

a:Animal⊢isCat(a)=True(isCat~(a))a : \mathsf{Animal} ⊢ \mathsf{isCat}(a) = \mathsf{True}(\widetilde{\mathsf{isCat}}(a))a:Animal⊢isCat(a)=True(isCat(a))

which says that by pulling back 'truth' along isCat~\widetilde{\mathsf{isCat}}isCat, we get a subobject equivalent to Cat\mathsf{Cat}Cat, and with the same indexing by Animal\mathsf{Animal}Animal. If you want to recover the 'whole space' as a type, rather than an indexed family of types, then you can use the sigma type to get something indexed by 111 instead of Animal\mathsf{Animal}Animal. That would be a type equivalent to Cat\mathsf{Cat}Cat.

You could also instead hypothesize isCat:Animal→Ω\mathsf{isCat} : \mathsf{Animal} → ΩisCat:Animal→Ω, which doesn't demand you to also invent Cat\mathsf{Cat}Cat directly, and instead allows you to define it. A lot of type theories arguably work this way, where everything is defined in terms of object classifiers (universes) whose values are implicitly usable as their corresponding types.

Dan Doel (Sep 14 2020 at 22:29):

I guess the point is that you aren't usually considering various families sliced over ΩΩΩ. You are reindexing from true:1↣Ω\mathsf{true} : 1 \rightarrowtail Ωtrue:1↣Ω in the slice over ΩΩΩ along the classifying map A→ΩA → ΩA→Ω.
