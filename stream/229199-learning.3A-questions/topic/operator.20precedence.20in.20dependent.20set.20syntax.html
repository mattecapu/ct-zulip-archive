Topic: operator precedence in dependent set syntax

Nathaniel Virgo (Mar 17 2025 at 17:16):

Does anyone know a good "cheat sheet" kind of document for the conventions in type theory / dependent set syntax? e.g. I can never remember whether ∑a:ABa→X \sum_{a: A} B_a\to X ∑a:A​Ba​→X means (∑a:ABa)→X \left(\sum_{a: A} B_a\right)\to X (∑a:A​Ba​)→X or ∑a:A(Ba→X) \sum_{a: A} \left(B_a \to X\right) ∑a:A​(Ba​→X) and I'm hoping to find a concise document where I can easily look that sort of thing up. I guess it mostly doesn't matter exactly which type theory we're talking about as this is mostly for my own informal notes.

Kenji Maillard (Mar 17 2025 at 21:01):

As a rule of thumb, I would have all operators binding a variable scope as far as possible on the right. So ∑a:ABa→X \sum_{a: A} B_a\to X ∑a:A​Ba​→X means ∑a:A(Ba→X) \sum_{a: A} \left(B_a \to X\right) ∑a:A​(Ba​→X).

Mike Shulman (Mar 18 2025 at 16:25):

I don't think there is a universal convention about this. When mixing Σ\SigmaΣ with →\to→ I always put parentheses.

Mike Shulman (Mar 18 2025 at 16:27):

Type theorists tend to follow the rule Kenji mentioned, but I think this is pretty unintuitive for mathematicians, who would probably not read ∑i=1nk+1\sum_{i=1}^n k + 1∑i=1n​k+1 as ∑i=1n(k+1)\sum_{i=1}^n (k + 1)∑i=1n​(k+1).

Mike Shulman (Mar 18 2025 at 16:29):

In the HoTT Book we followed the type-theorists' convention at least for Π\PiΠ, so that ∏a:ABa→X\prod_{a:A} B_a \to X∏a:A​Ba​→X means ∏a:A(Ba→X)\prod_{a:A} (B_a \to X)∏a:A​(Ba​→X), but this is probably the bit of notation in that book that is the most confusing to readers and I wouldn't do it again.

(Then again, if we had the whole book to write over again I would probably suggest using NuPRL/Agda syntax (a:A)→Ba→X(a:A) \to B_a \to X(a:A)→Ba​→X for dependent functions.)
