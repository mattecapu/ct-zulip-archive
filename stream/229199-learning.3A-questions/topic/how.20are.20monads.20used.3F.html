Over the last few days, the concept of a monad has (finally) started making some sense to me! It feels like this opens up a huge amount of new things that I can learn about now, which is exciting.

By the way, the resources that made things start to intuitively click for me were:

  • the introduction to monads in the context of probability monads in sections 1.1 of Paolo Perrone's thesis
  • the crash course on monads and their algebras in section 2.4 of "Higher-Dimensional Categories", by Eugenia Cheng and Aaron Lauda.

At this point, I'm curious to learn what people use monads for, at a very high level! Here are some of the examples I've seen so far:

  • We can use monads to talk about relationships between "iterated versions" of some kind of construction. The example I've read about here is that we can use monads to talk about probability distributions of probability distributions. The monad endofunctor PPP sends an object XXX to an object PXPXPX describing probability distributions on XXX. Interestingly, the natural transformation μ:PP→P\mu: PP \to Pμ:PP→P with components μX:PPX→PX\mu_X: PPX \to PXμX​:PPX→PX sends each "probability distribution of probability distributions" to a single probability distribution, I think intuitively obtained by taking a probability-weighted average.
  • We can use monads to convert objects into new objects that contain formal combinations of the parts of the original object. Then (at least sometimes) we can define an algebra for this monad that takes these formal combinations and returns parts of the original object. For example, I think there is a monad with endofunctor T:Set→SetT: \mathsf{Set} \to \mathsf{Set}T:Set→Set that sends a set to the underlying set of the free monoid on that set. For example, T{a,b,c}T\{a,b,c\}T{a,b,c} is a set that contains elements like abbcabbcabbc and babcbabcbabc. Then a (structured enough) function T{a,b,c}→{a,b,c}T\{a,b,c\} \to \{a,b,c\}T{a,b,c}→{a,b,c} is like a "multiplication rule" that takes in ordered tuples of elements from {a,b,c}\{a,b,c\}{a,b,c} and returns an element from {a,b,c}\{a,b,c\}{a,b,c}. As another cool example, we can send a directed graph GGG to the underlying graph of the free category induced by GGG. Intuitively, this augments GGG so that it has a directed edge for every tuple of "composable directed edges".
  • We can use monads to augment objects by adding in some constant additional data to each object. For example, in the maybe monad our endofunctor (−)∗:Set→Set(-)_*: \mathsf{Set} \to \mathsf{Set}(−)∗​:Set→Set adds an additional point to each set. So a set SSS gets augmented to the set S∐{∗}S \coprod \{*\}S∐{∗}. I would guess that we can generalize this by taking coproducts with any fixed object.
  • Once we have defined a monad with endofunctor TTT, we can consider morphisms of the form f:X→TYf:X \to TYf:X→TY (called "Kleisli morphisms"), which map from one of our original objects to one of our fancier objects that we've formed by using our monad endofunctor. These let us consider "hybrid" relationships between objects at different "tiers" of application of TTT. For example, if T:Set→SetT: \mathsf{Set} \to \mathsf{Set}T:Set→Set is the powerset endofunctor, then a function f:X→TYf: X \to TYf:X→TY maps each element of XXX to some subset of YYY. Or if TTT sends each set to the set of probability distributions on that set, then a function f:X→TYf: X \to TYf:X→TY associates to each element of XXX a probability distribution on YYY. Apparently, we can form a new category with our original objects, but where the morphisms are these fancier Kleisli morphisms.

Broadly, what are some other ways in which people use monads?

Patrick Nicodemus (Sep 21 2023 at 17:52):

Monads are used in homological algebra because they contain the necessary data to carry out the "bar construction."

As a first approximation, you can think of homological algebra as being an approach to algebra which allows us to embed ordinary algebra of groups, rings and modules into the broader universe of topological rings, groups and modules. Sometimes you can then replace your complicated group with a topological group which is simpler in some regards.

More concretely, say you have an Abelian group GGG which has lots of torsion in it which makes it hard to study. Homological algebra suggests trying to replace GGG with a similar "topological group" (a simplicial complex with a group structure) G′G'G′ which is equivalent to GGG in some respects but its group complexity is spread out and stratified through the dimensions of G′G'G′. So G′G'G′ has vertices, edges, faces and so on, its vertices form a group G0′G'_0G0′​, its edges form a group G1′G'_1G1′​, its faces form a group G2′G_2'G2′​, and so on, and these operations are all compatible with each other. The important thing is that each of the individual groups G0′,G1′,G2′G_0',G_1',G_2'G0′​,G1′​,G2′​ should all be relatively simple and better behaved/easy to understand than GGG. For example they should all be free Abelian groups. Thus in some sense we're trading off algebraic complexity for the geometric complexity of having a space spread out across multiple dimensions.

Monads can be used in an algorithm called the bar construction which associates to each algebraic object a "simplicial complex" of algebras which are free in each dimension. They are fundamental to homological algebra and many basic concepts of homological algebra can be understood in terms of monads.

Chris Grossack (they/them) (Sep 22 2023 at 04:08):

To every adjunction L⊣RL \dashv RL⊣R, the composite RLRLRL is a monad! Indeed, in case of algebraic structures, like groups, the free-forgetful adunction UF:Set→SetUF : \mathsf{Set} \to \mathsf{Set}UF:Set→Set is exactly the monad describing the algebraic gadget which you've already mentioned. Now, why does it matter that adjunctions give rise to monads?

Chris Grossack (they/them) (Sep 22 2023 at 04:09):

Say you have a category C\mathcal{C}C and a functor R:C→DR : \mathcal{C} \to \mathcal{D}R:C→D. A very natural question is to try and understand the image of RRR. Well in nice cases, RRR admits a left adjoint L⊣RL \dashv RL⊣R, and this gives us something to work with!

Indeed, every object Rc∈DRc \in \mathcal{D}Rc∈D in the image of RRR is automatically an algebra for the monad $$RL : \mathcal{D} \to \mathcal{D}$! (Can you see why? As a hint, consider the counit ϵc:LRc→c\epsilon_c : LRc \to cϵc​:LRc→c). So if we're trying to understand the image of RRR we know that we can restrict attention to the RLRLRL-algebras in D\mathcal{D}D!

Chris Grossack (they/them) (Sep 22 2023 at 04:12):

This is already great, but we can do more! Indeed, every RcRcRc admits the structure of an RLRLRL-algebra, and so we can look at the category DRL\mathcal{D}^{RL}DRL of RLRLRL-algebras in D\mathcal{D}D. We know that C\mathcal{C}C has an adjunction L⊣RL \dashv RL⊣R over D\mathcal{D}D, and the category DRL\mathcal{D}^{RL}DRL also has a free-forgetful adjunction F⊣UF \dashv UF⊣U over D\mathcal{D}D. Moreover, we've just said that R:C→DR : \mathcal{C} \to \mathcal{D}R:C→D factors through DRL\mathcal{D}^{RL}DRL, and now we have more structure (the structure of RLRLRL-algebras) to help us understand the image of RRR.

Chris Grossack (they/them) (Sep 22 2023 at 04:15):

There are general theorems which tell us when this lifted functor R:C→DRLR : \mathcal{C} \to \mathcal{D}^{RL}R:C→DRL is fully faithful (in which case RRR is called a descent morphism) or in the best case when RRR is an equivalence (in which case we call it effective descent). These monadicity theorems are extremely important, since they let us understand the (potentially complicated) category C\mathcal{C}C in terms of its image in the (potentially simpler) category D\mathcal{D}D (as long as we only look at the RLRLRL-structured objects in D\mathcal{D}D).

Chris Grossack (they/them) (Sep 22 2023 at 04:19):

(also as an aside, adjunctions are incredibly useful, and in some sense monads allow us to detect adjunctions. Indeed, we know that the composite RLRLRL will always be a monad. In fact the converse is also true -- every monad is of the form RLRLRL for some adjunction!

The kleisli category and the eilenberg-moore category give us two such "factorizations" for our monad! It turns out we have a category worth of factorizations, and the kleisli category is initial among the factorizations, while the eilenberg-moore category is terminal!)

Patrick Nicodemus (Sep 23 2023 at 13:37):

@David Egolf at this point it's useful to start synthesizing your comments with what Chris pointed out, that adjunctions give rise to monads. For any adjunction (L, R), RL is a monad, and you can choose the unit of the monad to be the unit ι:1→RL\iota : 1\to RLι:1→RL of the adjunction, and the multiplication μ:RLRL→RL\mu : RLRL\to RLμ:RLRL→RL would be the natural transformation RεLR\varepsilon LRεL.
Dually, LRLRLR is a comonad, with counit ε\varepsilonε given by the counit of the adjunction, and comultiplication δ\deltaδ given by LιRL\iota RLιR.
In the notes by Derek Wise you posted, all the important morphisms in those pictures are built out of not just the unit and counit of the adjunction but more specifically the maps ε\varepsilonε, LιRL\iota RLιR, and by repeated application of the functor LRLRLR. For this reason I claim that the picture could be also written, after a change of notation, using only the symbols ε\varepsilonε, δ=LιR\delta=L\iota Rδ=LιR, and G=LRG=LRG=LR. The theorem "every adjunction can be used to construct simplicial objects" can be broken down into two parts as "every adjunction gives rise to a comonad, and every comonad can be used to construct simplicial objects".

Patrick Nicodemus (Sep 23 2023 at 13:41):

So your question "is there an adjunction which can be used to produce the face and degeneracy maps between simplices?"
