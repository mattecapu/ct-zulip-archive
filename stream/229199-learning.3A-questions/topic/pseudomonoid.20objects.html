Topic: pseudomonoid objects

Beppe Metere (Oct 14 2022 at 06:35):

Let C\mathcal{C}C be a category with finite products. One can define a monoid object internal to C\mathcal{C}C as an object MMM of C\mathcal{C}C endowed with identity 1→M1\to M1→M and multiplication M×M→MM\times M\to MM×M→M satisfying a bunch of diagrams. However, one can also define a monoid object in a representable fashion: a functor F ⁣:Cop→MonF\colon \mathcal{C}^{op}\to \mathbf{Mon}F:Cop→Mon such that the composite with the forgetful functor Mon→Set \mathbf{Mon}\to \mathbf{Set} Mon→Set gives the representable functor hMh_MhM​.

It seems to me that the "same" can be done in dimension 2. Am I right? I mean, a pseudomonoid object in a 2-category is usually defined as an object with structure satisfying some conditions, which are expressed as diagrams filled with coherent isomorphisms. Should one also define them as pseundofunctors to MonCat \mathbf{MonCat}MonCat that composed with the forgetful 2-functor MonCat→Cat \mathbf{MonCat}\to \mathbf{Cat} MonCat→Cat give 2-representables?

Did anyone ever take this point of view?
