Topic: definition of "path" and principle of equivalence

David Egolf (May 06 2023 at 17:58):

I learned a little about the principle of equivalence recently, and in that context I was thinking about the definition of a path. I'm hoping to understand better, through this example, when picking a specific object among isomorphic objects is fine, and when we'd prefer to not make such a choice.

We might define "a path in XXX" to be a continuous function f:I→Xf: I \to Xf:I→X, where I=[0,1]I = [0,1]I=[0,1] has the usual subspace topology and XXX is some topological space. Accepting this definition, the following sentence P(I)P(I)P(I) is a true statement about III: "A continuous function from III to XXX is a path in XXX".

Now, let JJJ be some topological space that is isomorphic (homeomorphic) to III in Top\mathsf{Top}Top, with J≠IJ \neq IJ=I. Using the definition above, the following sentence P(J)P(J)P(J) about JJJ is false: "A continuous function from JJJ to XXX is a path in XXX".

So, we have I≅JI \cong JI≅J, but the truth values of P(I)P(I)P(I) and P(J)P(J)P(J) are not equal or even isomorphic in some appropriate partially ordered set of truth values. Does this violate the principle of equivalence?

Motivated by this, I am wondering if it would be more in the spirit of the principle of equivalence to define "a path in XXX" like this: a path in XXX is a continuous function f:J→Xf: J \to Xf:J→X, where JJJ is some topological space isomorphic to the unit interval III. Is this an approach that people sometimes take?

Any thoughts on these specific questions, or more generally on the topic of selecting a specific object when isomorphic alternatives exist, would be most welcome.

Yes. More objectively - since "principles" are not the same as mathematical facts, and it's often nice to state facts - we can say that you have defined a map PPP from the class of topological spaces to the set of truth values, but this does not extend to a functor from the category of topological spaces to the poset of truth values, since it maps isomorphic topological spaces to nonisomorphic topological spaces.

Verity Scheel (May 07 2023 at 00:13):

It depends on what you mean by “is” and ===. There are a bunch of angles to tackle this question.

For instance, in HoTT, there simply is not any JJJ that is isomorphic to III but is not equal to III, since equality and homeomorphism coincide for topological spaces.

Additionally, if you take “is” to mean === in HoTT, the proposition “A continuous function from III to XXX is a path in XXX” is true for all spaces homeomorphic to III as well. But I generally think that you want definitions to be something stricter than propositional equality (pardon the Type Theory jargon) ...

Neglecting HoTT for a moment, what do we care about in the choice of I=[0,1]I = [0,1]I=[0,1]? I think you would agree that we don't care about the specific numbers: [−1,1][-1, 1][−1,1] would work just as well, [−42,132.516][-42, 132.516][−42,132.516] too. But I think being _merely_ isomorphic is too weak: we do want to be able to work with the endpoints, to say that a path starts at a known point and ends at a known point. If you don't know what the isomorphism is, you cannot say which point is the start and which the end, so I might suggest a tweak to your second definition that makes it remember the end points at least.

You can make my last point more formal by talking about path induction and stuff in HoTT, but I don't want to type out the details at the moment. It's the difference between ΣJ,(J=I)∧P(J)\Sigma J, (J = I) \wedge P(J)ΣJ,(J=I)∧P(J) (which is provably equal to P(I)P(I)P(I)) and ΣJ,∥J=I∥∧P(J)\Sigma J, \| J = I \| \wedge P(J)ΣJ,∥J=I∥∧P(J) (which is not, due to the propositional truncation of the equality).

Verity Scheel (May 07 2023 at 00:39):

If you want “is” to mean a definitional equality, my first comment would be that it's customary to write the thing you are defining on the left: “A path in XXX is a continuous function from III to XXX”.

From that point of view, it is true that if you define path that way, then other equalities fail to hold definitionally, like “A path in XXX is a continuous function from [−42,132.516][-42, 132.516][−42,132.516] to XXX” fails to hold, since III is not definitionally equal to [−42,132.516][-42, 132.516][−42,132.516].

But that is at the meta-theoretic level: within the logic you are working in, you cannot observe a “lack of definitional equality”. It just doesn't make sense to ask that. (Such as for the reason that we know that definitional equality is an under-approximation of equality in the theory.) This is the difference between “judgment” and “proposition”, where that distinction makes sense. Propositions can be negated, judgments cannot (except at a meta-theoretic level).

Returning to this:

Fix a topological space XXX. Consider the set of triples (J,h,p)(J, h, p)(J,h,p) where JJJ is a topological space, h:I≅Jh : I \cong Jh:I≅J is a specified homeomorphism between JJJ and III, and p:J→Xp : J \to Xp:J→X is a continuous function from JJJ to XXX.

If you want to recover something that is equivalent to the usual definition of path you can do this: Define an equivalence relation on this set by (J1,h1,p1)∼(J2,h2,p2)(J_1, h_1, p_1) \sim (J_2, h_2, p_2)(J1​,h1​,p1​)∼(J2​,h2​,p2​) iff p1∘h1=p2∘h2p_1 \circ h_1 = p_2 \circ h_2p1​∘h1​=p2​∘h2​. This is quite literally saying that the only thing that matters is the view of these paths via the usual definition I→XI \to XI→X.

You can have finer or coarser equivalence relations as well.

One nice thing about allowing different isomorphs of [0,1][0,1][0,1] is that you can sometimes get stricter behavior. For instance, a Moore path in XXX is a continuous map [0,r]→X[0,r] \to X[0,r]→X for some real number r≥0r\ge 0r≥0. Then if p:[0,r]→Xp:[0,r] \to Xp:[0,r]→X and q:[0,s]→Xq:[0,s] \to Xq:[0,s]→X are Moore paths with p(r)=q(0)p(r) = q(0)p(r)=q(0), we define their concatenation t:[0,r+s]→Xt:[0,r+s] \to Xt:[0,r+s]→X by t(x)=p(x)t(x)=p(x)t(x)=p(x) if x≤rx\le rx≤r and t(x)=q(x−r)t(x) = q(x-r)t(x)=q(x−r) if x≥rx\ge rx≥r.
