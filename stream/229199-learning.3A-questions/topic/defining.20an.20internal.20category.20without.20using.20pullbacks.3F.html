A monoid involving a set MMM requires some multiplication function μ:M×M→M\mu: M \times M \to Mμ:M×M→M. Noticing this, we can be inspired to define a monoid internal to any category with products. Our multiplication for a monoid involving an object MMM will be a morphism μ:M×M→M\mu: M \times M \to Mμ:M×M→M. However, we can generalize further and define a monoid in any monoidal category! Then the "multiplication" for a monoid involving an object MMM in a monoidal category CCC involves a morphism μ:M⊗M→M\mu: M \otimes M \to Mμ:M⊗M→M. Here ⊗:C×C→C\otimes: C \times C \to C⊗:C×C→C is the functor that is part of the data that makes CCC a monoidal category.

So, although our initial definition of a monoid involved a morphism from a product, it turns out we can define monoids internal to monoidal categories.   1. Obj(C)\mathsf{Obj}(\mathcal{C})Obj(C) denotes the class of objects of C\mathcal{C}C;

  2. Desc(C)\mathsf{Desc}(\mathcal{C})Desc(C) denotes the class of reflexive 1-graphs in C\mathcal{C}C, where

    • we define a 1-graph as a pair (s,t):A⇉B(s,t):A \rightrightarrows B(s,t):A⇉B of arrows in C\mathcal{C}C
    • and we say that this 1-graph is reflexive if it is equipped with an arrow i:B→Ai:B \to Ai:B→A in C\mathcal{C}C for which the equations s∘i=idB=t∘is \circ i = \mathsf{id}_B = t \circ is∘i=idB​=t∘i hold.
