Can a Hom-functor be “reversed”?

It seems you're trying to ask the following question: let CCC be a category and assume there is an object aaa such that hom⁡(a,−):C→Set\hom(a,-) : C \to Sethom(a,−):C→Set is an equivalence of categories (so applying the inverse you indeed "get xxx back out of hom⁡(a,x)\hom(a,x)hom(a,x)"). What do I know about aaa? The answer is: this is a fairly restrictive assumption on aaa (and on CCC, of course, which ends up being essentially the category of sets). For example, a functor that is an equivalence of categories preserves all colimits, as well as all limits: then, hom⁡(a,−)\hom(a,-)hom(a,−) must preserve all colimits, making aaa into a [[tiny object]]. From an introductory point of view, what is useful about a functor being “representable” as a Hom-functor?

A conversation that might enlighten you on this matter is happening right now in the "classifying object" thread. Very simply put: if a functor FFF is representable, it acts like hom⁡(a,−)\hom(a,-)hom(a,−) for some (uniquely determined) a∈Ca\in Ca∈C.
