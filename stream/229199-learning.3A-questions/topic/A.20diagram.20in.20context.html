Topic: A diagram in context

Fawzi Hreiki (Sep 27 2020 at 22:53):

At first approximation, a mathematical structure SSS in a category C\mathscr{C}C consists of:

  • Data/structure DDD (certain objects and morphisms in C\mathscr{C}C).
  • Axioms/properties AAA (the commutativity of certain diagrams, usually in C\mathscr{C}C but sometimes elsewhere).

For example, a monoid MMM in a category C\mathscr{C}C with products consists of objects and maps 1→eM←.M×M1 \xrightarrow{e} M \xleftarrow{.} M \times M1e​M.​M×M such that the usual associativity and identity diagrams commute.

So I want to say that a structure SSS in C\mathscr{C}C consists of:

  • A (small) category KKK and a functor D:K→CD: K \rightarrow \mathscr{C}D:K→C.
  • A (small) category JJJ and a functor A:J→CA: J \rightarrow \mathscr{C}A:J→C (lets keep the axioms in C\mathscr{C}C for simplicity)

We can package all the data into one functor by taking coproducts in Cat\textbf{Cat}Cat, and likewise for the axioms. Usually KKK has some further categorical structure (e.g. products, power objects, etc..) which DDD is required to preserve, and likewise for JJJ and AAA.

The problem I'm having is that the functor AAA is meant to be able to reference the functor DDD, so AAA needs to be a functor into C\mathscr{C}C 'in context DDD'. So either AAA needs to be from JJJ into some sort of comma category relating DDD and C\mathscr{C}C or it needs to be from some sort of exponential category relating JJJ and KKK into C\mathscr{C}C. I'm sure theres an easy fix but its going over my head. Any help would be much appreciated.

Morgan Rogers (he/him) (Sep 28 2020 at 08:53):

Have you heard of sketches? They package data in approximately the way you're describing. The workaround is that the functor AAA you mention should have codomain KKK rather than C\mathcal{C}C, so the diagrams commute in KKK already (commutativity is preserved by any functor, in particular by DDD).
