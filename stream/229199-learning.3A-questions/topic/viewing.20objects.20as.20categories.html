Topic: viewing objects as categories

David Egolf (Sep 19 2022 at 15:53):

Some objects can be viewed as categories themselves, for example:

  • Each set can be viewed as a discrete category
  • Each monoid can be viewed as a category with one object
  • Each group can be viewed as a category with one object, where all morphisms are isomorphisms
  • Each groupoid can be viewed as a category where each morphism is an isomorphism
  • Each preorder can be viewed as a category, with up to one morphism between each pair of objects

However, is there a way to view some functors as categories? Functors can be viewed as objects, in categories where functors are the objects and the morphisms are natural transformations. However, I don't know how to "zoom in" on these objects and view them as categories in their own right.

If we could view functors as categories, then we could view a lot of things as categories, including functions and group representations. If this isn't possible to do this with all functors, is it maybe possible to view some special functors as categories?

John Baez (Sep 19 2022 at 17:44):

To unpack Mike's answer a little bit: if you're trying to view functors as categories, David, you should start by "decategorifying" and try to view functions as sets. Any function f:X→Yf: X \to Yf:X→Y gives a set called its graph:

{(x,y)∈X×Y  ∣  y=f(x)} \{(x,y) \in X \times Y \; | \; y = f(x) \} {(x,y)∈X×Y∣y=f(x)}

and in fact in some axiomatic approaches to math we say a function from XXX to YYY is its graph: any subset of X×YX \times YX×Y with certain well-known properties counts as a function from XXX to YYY.

John Baez (Sep 19 2022 at 17:47):

You might try to copy this idea and define the graph of a functor F:C→DF: C \to DF:C→D to be some subcategory of C×DC \times DC×D.

John Baez (Sep 19 2022 at 17:49):

However, that's not quite right.

It turns out that what we get is not a subcategory of C×DC \times DC×D, but a category equipped with a functor to Cop×DC^{\text{op}} \times DCop×D. So there are two subtleties: replacing CCC with its opposite, and replacing "subcategory of...." with "category with a functor to...".

Mike Shulman (Sep 20 2022 at 17:42):

Right! For a general category, you often want to consider generalized elements whose domains (sometimes called "stages") are not completely arbitrary, but more general than the terminal object. Specifically, you want to consider a set of domain objects that "generate" the category in some sense. There are a bunch of such senses; the weakest is a [[separator]] and the strongest is a [[dense subcategory]]. The singleton {1}\{1\}{1} in Set\rm SetSet, and the collection of representables in a presheaf category, are both dense; but the singleton {T}\{T\}{T} in a functor category is not in general even a separator.
