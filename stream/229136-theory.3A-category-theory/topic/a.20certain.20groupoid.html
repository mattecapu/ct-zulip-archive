Reid Barton (Oct 26 2020 at 15:15):

The category C\mathcal{C}C is equivalent to the set N\mathbb{N}N viewed as a discrete category. More specifically, the functor sending any S⊂{1,…,n}S \subset \{1, \ldots, n\}S⊂{1,…,n} to nnn is an equivalence of categories. We could think of C\mathcal{C}C as a version of N\mathbb{N}N where each n∈Nn \in \mathbb{N}n∈N has been "blown up" into a contractible groupoid.

Clearly, there must be some reason that you need this "blown up" C\mathcal{C}C, otherwise you'd just use N\mathbb{N}N instead. A common reason in homotopy theory/higher category theory is that you need to take a cofibrant replacement in order to make some 1-categorical construction be homotopy/equivalence-invariant. For example, consider the associator in a monoidal category. We want to say that in the expression x⊗y⊗zx \otimes y \otimes zx⊗y⊗z, it's "the same" whether you consider it as (x⊗y)⊗z(x \otimes y) \otimes z(x⊗y)⊗z or as x⊗(y⊗z)x \otimes (y \otimes z)x⊗(y⊗z). But to ask for a literal equality of objects would be "evil"--not equivalence-invariant. So, instead of having a single operation {(12)3=1(23)}\{(12)3 = 1(23)\}{(12)3=1(23)}, we "blow up" the possible ways to evaluate x⊗y⊗zx \otimes y \otimes zx⊗y⊗z to the equivalent groupoid {(12)3↔∼1(23)}\{(12)3 \xleftrightarrow{\sim} 1(23)\}{(12)3∼​1(23)}. The interpretation of this isomorphism is as the associator αx,y,z:(x⊗y)⊗z↔∼x⊗(y⊗z)\alpha_{x,y,z} : (x \otimes y) \otimes z \xleftrightarrow{\sim} x \otimes (y \otimes z)αx,y,z​:(x⊗y)⊗z∼​x⊗(y⊗z). In general, for each nnn, we have a contractible groupoid of ways to interpret the expression x1⊗⋯⊗xnx_1 \otimes \cdots \otimes x_nx1​⊗⋯⊗xn​, and what forces us to use these fancy blown-up contractible groupoids is the operad structure which relates these operations for varying nnn.

John Baez (Oct 26 2020 at 21:01):

Quite generally, Tim, you'll find category theorists puzzled if you declare yourself unwilling to think of equivalent categories as "the same". You can always take a category C and find an equivalent category called a "skeleton" which has one object for each isomorphism class of objects in C. In this skeleton isomorphic objects are equal. So, when you say something like

I don’t want to think of isomorphic objects as being equal

then category theorists start wondering what you're up to.
