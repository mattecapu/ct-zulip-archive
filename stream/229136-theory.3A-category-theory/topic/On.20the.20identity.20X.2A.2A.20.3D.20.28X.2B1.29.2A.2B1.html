Sofia B (Apr 14 2026 at 18:00):

There is an observation that

(A+B)∗=(A∗B)∗A∗(A + B)^\ast = (A^*B)^\ast A^\ast (A+B)∗=(A∗B)∗A∗

that is, you group all the As to the left of each B together as well as the leftovers at the end.

Anyway, one can then see that from this is follows that

(A+1)∗=(A∗1)∗A∗=(A∗)∗A∗ (A + 1)^\ast = (A^* 1)^\ast A^\ast = (A^*)^* A^* (A+1)∗=(A∗1)∗A∗=(A∗)∗A∗

Then if you add 1 to this, you get

1+(A+1)∗=1+(A∗)∗A∗=(A∗)∗1 + (A+1)^* = 1 + (A^*)^* A^* = (A^*)^*1+(A+1)∗=1+(A∗)∗A∗=(A∗)∗

(the last step uses the 'refolding' iso 1+A∗A=A∗1 + A^* A = A^*1+A∗A=A∗)

Sofia B (Apr 16 2026 at 11:40):

This observation that you can group things like this is presumably ancient, but it comes up in the theory of Star height for regular languages (the star height problem is: what is the regular expression for a regular language with the least number of nested *s?)

However, star-height is not based on isomorphisms: as regular languages, $$A^\ast^\ast = A^\ast$$ but these are not isomorphic; the morphism going to the right 'forgets' the internal list boundaries

(one could technically make a bijection here, but that is not connected to the regular-language-equivalence that is used in the theory of star height. Language equivalence is like taking the multivariate polynomial functor from an expression like (A+B)∗A(A+B)^\ast A(A+B)∗A and truncating the index set so there is only one index per (ordered) monomial made up of A, B, etc - aka, a language is just a disjunction of its strings)

As for proving the identity, we can write a Haskell function for the grouping:

group :: [Either a b] -> ([([a], b)], [a])
group xs = go xs ([], [])
  where
    go :: [Either a b] -> ([([a], b)], [a]) -> ([([a], b)], [a])
    go [] r = r
    go (Left a : xs) (lgs, ros) = go xs (lgs, a : ros)
    go (Right b : xs) (lgs, ros) = go xs ((ros, b) : lgs, [])

from which we can extract the proof

(A+B)∗=1+(A+B)∗(A+B)=1+(A∗B)∗A∗(A+B)=1+(A∗B)∗(A∗A+A∗B)=1+(A∗B)∗(A∗B)+(A∗B)∗(A∗A)=(A∗B)∗+(A∗B)∗A∗A=(A∗B)∗(1+A∗A)=(A∗B)∗A∗(A + B)^\ast \\ = 1 +(A + B)^\ast (A + B) \\ = 1 + (A^\ast B)^\ast A^\ast (A + B) \\ = 1 + (A^\ast B)^\ast (A^\ast A + A^\ast B) \\ = 1 + (A^\ast B)^\ast (A^\ast B) + (A^\ast B)^\ast (A^\ast A) \\ = (A^\ast B)^\ast + (A^\ast B)^\ast A^\ast A \\ = (A^\ast B)^\ast (1 + A^\ast A) \\ = (A^\ast B)^\ast A^\ast(A+B)∗=1+(A+B)∗(A+B)=1+(A∗B)∗A∗(A+B)=1+(A∗B)∗(A∗A+A∗B)=1+(A∗B)∗(A∗B)+(A∗B)∗(A∗A)=(A∗B)∗+(A∗B)∗A∗A=(A∗B)∗(1+A∗A)=(A∗B)∗A∗

We use the identity we are proving recursively, but this can be justified in that we are expanding the set and then using the identity on a smaller instance, and the instance size is well-founded.

Still, it's not so easy to find this... (and the proof technique is a bit non-obvious)

Maybe a better proof is that the list is the free monoid, and so we can interpret (A+B)∗(A+B)^\ast(A+B)∗ homomorphically in the monoid over (A∗B)∗A∗(A^\ast B)^\ast A^\ast(A∗B)∗A∗

(Monoid interpretations are just folds, which the above code is an explicit form of)

Anyway, the monoid structure is

unit = ([], [])

(groups1, as) ⋅ ([], as') = (groups1, (as ++⋅ as'))
(groups1, as) ⋅ (((ps, b) : groups2), as') = (groups1 ++ [(as ++ ps, b)] ++ groups 2, as')

If you will forgive the pseudoHaskell.
