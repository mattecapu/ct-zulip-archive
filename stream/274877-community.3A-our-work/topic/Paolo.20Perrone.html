Tom Leinster (Oct 12 2024 at 14:20):

An underused piece of notation (in my opinion) is m:a↛bm: a \not\to bm:a→b to mean m∈M(a,b)m \in M(a, b)m∈M(a,b), where M:Aop×B→SetM: \mathbf{A}^{\mathrm{op}} \times \mathbf{B} \to \mathbf{Set}M:Aop×B→Set is a functor. (Here ↛\not\to→ should be a profunctory crossed arrow, which I don't know how to typeset here.) What's good about this notation is that you can compose like this:

a′→fa↛mb↦a′↛mfba' \stackrel{f}{\to} a \stackrel{m}{\not\to} b \qquad\mapsto\qquad a' \stackrel{mf}{\not\to} ba′→fa→m​b↦a′→mf​b

a↛mb→gb′↦a↛gmb′a \stackrel{m}{\not\to} b \stackrel{g}{\to} b' \qquad\mapsto\qquad a \stackrel{gm}{\not\to} b'a→m​b→gb′↦a→gm​b′

Everything works particularly nicely if you write M:Aop×B→SetM: \mathbf{A}^{\mathrm{op}} \times \mathbf{B} \to \mathbf{Set}M:Aop×B→Set as M:A↛BM: \mathbf{A} \not\to \mathbf{B}M:A→B, as then the elements of M:A↛BM: \mathbf{A} \not\to \mathbf{B}M:A→B look like m:a↛bm: a \not\to bm:a→b. This is the less popular of the two conventions for (bi)modules/profunctors/distributors, but it has this advantage.

Drawing elements of presheaves as arrows is a special case of this notation.

I used this notation for bimodule elements in my work on self-similarity, e.g. here, but I wouldn't be remotely surprised if others had done the same before me.

John Baez (Oct 12 2024 at 19:17):

I tend to use ⇸ for loose arrows in a double category - like relations, spans and profunctors. I imagine the bar as negation, like as "you think this is a tight arrow? No, it's a loose arrow?"

Then the use of f:a⇸b to denote an element of a profunctor F:A⇸B would be a nice case of the microcosm principle: "to make sense a morphism from an object of one category to an object of another, we need a profunctor from one category to another". Jim Dolan might say f rides the profunctor F, like a person rides a horse, or a map between monoid objects in two different monoidal categories rides a lax monoidal functor between these monoidal categories.

Matteo Capucci (he/him) (Oct 22 2024 at 09:43):

I might add myself to the list of people who have been using this idea for a while, although never in (published) writing. Specifically you've seen me doing that if we ever talked about categorical systems theory, where the main structure of interest are modules over double categories, which are the double version of modules over categories, ie (co)presheaves.
Thus given such a module Sys\bf SysSys over the double category C\mathbb{C}C, I usually draw elements of Sys(A){\bf Sys}(A)Sys(A) as bubbles with a dangling wire named AAA, to which (loose) arrows of C\mathbb{C}C can be attached (and I draw them as boxes). 'Functoriality' is then the rule that a bubble S∈Sys(A)S \in {\bf Sys}(A)S∈Sys(A) can absorb any arrow p:A↛Bp:A \nrightarrow Bp:A↛B to become a bubble S⋅p∈Sys(B)S \cdot p \in {\bf Sys}(B)S⋅p∈Sys(B).

Let me try to answer.

  • You mention the statement of the yoneda lemma around page 16. Have you thought about how one might go about proving yoneda in this visualisation?

I haven't thought about that, but I guess the proof would just be a translation of the ordinary one. Or do you have an idea for a somewhat different proof? (That would be quite interesting.)

  • I'm not as familiar with weighted limits, but do you know if there's a nice way to interpret ordinary limits/colimits in this picture?

Yes sure! Those are exactly the weighted limits for which the weighting is 1, that is, for which all the cones have exactly one arrow to each object. This recovers the standard intuition for limits as well.
Same for colimits. (If you need some intuition for that, if you like my style you might want to check section 3 of my lecture notes, but there's plenty of other material as well, such as Chapter 5 of Tom Leinster's book.)

  • Would it be possible to simultaneously add both presheaves and copresheaves to C as two kinds of "virtual objects"? Or must you focus on only one at a time?

Good question. I don't know, maybe using profunctors. I guess it strongly depends on what you want to do in the resulting category, do you have an application in mind?

  • How might one visualise adjunctions in this picture?

Adjunctions are quite nice in this picture.
Basically, what's going on is the following (let's take right-adjoints as an example, the picture for left-adjoints is dual). If F : C -> D has a right-adjoint, it means that for every object D of D, the presheaf D(F-,D): C^op -> Set is representable. That is:

  • First of all, the virtual arrows of the presheaf, out of an object C of C, are real arrows FC -> D of D, similarly to what happened in pointwise Kan extensions (section 3.5 of my paper).
  • Now the adjunction condition says that these virtual arrows are actually real arrows of C, and they go to an object GD.
  • If you do that for every D of D, you get arrows, again similarly to what happened for Kan extensions.

This is quite well explained in the work of David Ellerman, see the references in my paper.
