Mitchell Riley (Apr 18 2021 at 01:57):

A while ago I spotted something about learners; I'll record it here in the chance someone finds it useful.

The 'Backprop as Functor' paper defines learners A↛BA \nrightarrow BA↛B as tuples (P,I:P×A→B,U:P×A×B→P,r:P×A×B→A)(P, I : P \times A \to B, U : P \times A \times B \to P, r : P \times A \times B \to A)(P,I:P×A→B,U:P×A×B→P,r:P×A×B→A), up to an equivalence: two learners are equivalent if there is a bijection P→P′P \to P'P→P′ so that the components of the two learners are equal up to this map. They mention that you might want to consider learners equal if there's just a surjection P→P′P \to P'P→P′, but if you go all the way to considering learners equivalent if there's any map P→P′P \to P'P→P′ that commutes with the components, something interesting happens.

This relation is now exactly a coend over SetSetSet, and the set of learners A↛BA \nrightarrow BA↛B is given by

∫PSet(P×A,B)×Set(P×A×B,P)×Set(P×A×B,A)\int^{P} Set(P \times A, B) \times Set(P \times A \times B, P) \times Set(P \times A \times B, A)∫PSet(P×A,B)×Set(P×A×B,P)×Set(P×A×B,A)

Learners and lenses are closely related, and we can use the same old moves to write the body of the coend in an 'optic' style:

∫PSet(P×A,B)×Set(P×A×B,P)×Set(P×A×B,A)≅∫PSet(P×A,B)×Set(P×A×B,P×A)≅∫P,QSet(P×A,Q)×Set(P×A,B)×Set(Q×B,P×A)≅∫P,QSet(P×A,Q×B)×Set(Q×B,P×A)\begin{aligned} &\int^{P} Set(P \times A, B) \times Set(P \times A \times B, P) \times Set(P \times A \times B, A) \\ & \cong \int^{P} Set(P \times A, B) \times Set(P \times A \times B, P \times A) \\ & \cong \int^{P, Q} Set(P \times A, Q) \times Set(P \times A, B) \times Set(Q \times B, P \times A) \\ & \cong \int^{P, Q} Set(P \times A, Q \times B) \times Set(Q \times B, P \times A) \end{aligned}​∫PSet(P×A,B)×Set(P×A×B,P)×Set(P×A×B,A)≅∫PSet(P×A,B)×Set(P×A×B,P×A)≅∫P,QSet(P×A,Q)×Set(P×A,B)×Set(Q×B,P×A)≅∫P,QSet(P×A,Q×B)×Set(Q×B,P×A)​

I think this symmetrical description is quite nice. There's now an obvious involution Learn→LearnopLearn \to Learn^{op}Learn→Learnop: just switch the two functions in the coend! Written out using the original definition of a learner, the 'adjoint' of a A↛BA \nrightarrow BA↛B given by (P,I,U,r)(P, I, U, r)(P,I,U,r) is a learner B↛AB \nrightarrow AB↛A with parameter set P×AP \times AP×A, and

I†:(P×A)×B→AI†(p,a,b)=r(p,a,b)U†:(P×A)×B×A→(P×A)U†(p,a′,b,a)=(U(p,a′,b),a)r†:(P×A)×B×A→Br†(p,a′,b,a)=I(U(p,a′,b),a)\begin{aligned} I^\dagger : (P \times A) \times B &\to A \\ I^\dagger(p, a, b) &= r(p, a, b) \\ U^\dagger : (P \times A) \times B \times A &\to (P \times A) \\ U^\dagger(p, a', b, a) &= (U(p, a', b), a) \\ r^\dagger : (P \times A) \times B \times A &\to B \\ r^\dagger(p, a', b, a) &= I(U(p, a', b), a) \end{aligned}I†:(P×A)×BI†(p,a,b)U†:(P×A)×B×AU†(p,a′,b,a)r†:(P×A)×B×Ar†(p,a′,b,a)​→A=r(p,a,b)→(P×A)=(U(p,a′,b),a)→B=I(U(p,a′,b),a)​

If you do this twice, you end up with a learner A↛BA \nrightarrow BA↛B with parameter set P×A×BP \times A \times BP×A×B, which is the original learner again, but running 'one training datum behind': when you give it a new bit of data, it updates PPP using the remembered data and stores the new A×BA \times BA×B for next time. To show this is equivalent to the original learner, we need a map P×A×B→PP \times A \times B \to PP×A×B→P that commutes with the components, and for this we have UUU.

I haven't kept up with the latest on learners, does anyone have a sense of whether this notion of equivalence is too strong to be useful?
