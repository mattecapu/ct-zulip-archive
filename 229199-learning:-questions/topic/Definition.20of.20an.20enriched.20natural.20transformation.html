The question is pretty well laid out on the link, but the main idea is: why does an enriched natural transformation use maps from the monoidal unit III, as opposed from some object P:VP : \mathcal{V}P:V?

Using objects that are different from the monoidal unit seems to be in line what enriched CT is all about, and the whole reasoning of using weighted limits. But Kelly's enriched CT book (and all other sources I could find) define it as a map from the monoidal unit.

Matteo Capucci (Nov 25 2020 at 12:54):

I want to point out that using III is not necessary to express naturality. Well, it is if you do it the way Kelly does, using the unitors of V\mathcal VV, but it is not if you just write down the equation expressing naturality.
What I mean here is that given two V\mathcal VV-functors F,G:C→DF, G : \mathbf C \to \mathbf DF,G:C→D, one can define a V\mathcal VV-natural transformation α:F→G\alpha:F \to Gα:F→G to be an obCob \mathbf CobC-indexed collection of morphisms αX:P→D(FX,GX)\alpha_X : P \to \mathbf D(FX, GX)αX​:P→D(FX,GX).
Then you ask naturality in this way: given f:Q→C(X,Y)f : Q \to \mathbf C(X,Y)f:Q→C(X,Y), you want

αX⊗(f;GX,Y);∘D=((f;FX,Y⊗αY);∘D\alpha_X \otimes (f;G_{X,Y}) ; \circ_D = ((f;F_{X,Y} \otimes \alpha_Y) ; \circ_DαX​⊗(f;GX,Y​);∘D​=((f;FX,Y​⊗αY​);∘D​

Matteo Capucci (Nov 25 2020 at 13:04):

Fixing PPP for all the αX\alpha_XαX​ seems unavoidable, since there would be no way to express naturality then (basically, the left and right hand of the above equation would have a different source!)
To relax this, one could go galaxy brain and use the end definition of natural transformations:
[F,G]=∫X∈CD(FX,GX)[F,G] = \int_{X \in \mathbf C} \mathbf D(FX,GX)[F,G]=∫X∈C​D(FX,GX)
Then one would add a weight W:C→VW : \mathcal C \to \mathcal VW:C→V and define WWW-weighted natural transformations to be
[F,G]W=∫X∈C[WX,D(FX,GX)][F,G]^W = \int_{X \in \mathbf C} [WX, \mathbf D(FX,GX)][F,G]W=∫X∈C​[WX,D(FX,GX)]

Matteo Capucci (Nov 25 2020 at 13:07):

Indeed, this is what one does to define co/limits in the enriched settings! So why not here?

I'm fairly sure putting all of them togheter saves their compositional structure: if you have natural transformations α:[F,G]\alpha : [F,G]α:[F,G] (weighted, say, by WWW) and β:[G,H]\beta : [G,H]β:[G,H] (weighted by TTT), you should be able to compose them by tensoring, obtaining βα:[F,H]\beta\alpha : [F,H]βα:[F,H] weighted by W⊗TW \otimes TW⊗T.
The identity natural transformation is I→1XC(X,X)→FD(FX,FX)I \overset{1_X}\to \mathbf C(X,X) \overset{F}\to \mathbf D(FX,FX)I→1X​C(X,X)→FD(FX,FX), and indeed it is the unit of the composition defined above

Reid Barton (Nov 25 2020 at 15:18):

I guess a more down-to-earth viewpoint is what's implicitly in the first few messages of the twitter thread: V\mathcal{V}V-functors between two V\mathcal{V}V-categories form a V\mathcal{V}V-category, and the morphisms of the underlying category are the enriched natural transformations.

Reid Barton (Nov 25 2020 at 15:21):

Whether looking at the underlying category of a V\mathcal{V}V-category is useful or not probably depends quite a lot on V\mathcal{V}V. For example, for V\mathcal{V}V = abelian groups it's quite a sensible thing to do.

Bruno Gavranovic (Nov 25 2020 at 22:19):

Perhaps the relevant distinction is that it is only sensible to consider individual enriched natural transformations in certain situations, whereas considering the collection of enriched natural transformations between two enriched functors is always sensible (for exactly the same reason that passing to the underlying category is only sensible in some settings).
